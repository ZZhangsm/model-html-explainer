1. 系统概述

OneRec / OpenOneRec 的核心思想是将推荐建模为一个统一的序列生成问题。传统推荐系统采用多级联范式（召回 → 粗排 → 精排），每级独立优化，上级准确度成为下级的瓶颈。OneRec 系列尝试用单一生成模型取代整个级联管线。

生产系统

2. OneRec 架构（论文 2502.18965）

OneRec 采用 Encoder-Decoder 结构：编码器读取用户历史行为序列，解码器逐步生成用户可能感兴趣的 Session 列表。结合稀疏 MoE（Mixture-of-Experts）扩展模型容量，在不成比例增加计算量的前提下显著提升用户兴趣建模能力。

Encoder
用户历史序列编码
→
Sparse MoE Transformer
扩展容量，控制 FLOPs
→
Decoder
自回归生成 Session 列表

特征工程

用户侧特征：正反馈历史行为序列 ℋu = {vh1, vh2, ..., vhn}，包含有效观看/点赞/关注/分享的视频。

每个视频 vi 使用多模态嵌入 ei ∈ ℝd 表示（Luo et al., 2024），这些嵌入与真实用户-物品行为分布对齐。

基于预训练的 RQ-VAE 模型将嵌入编码为语义 ID（Semantic IDs）：

e → RQ-VAE(e) = [s1, s2, ..., sL]
    # L = 3 层残差量化

最终用户历史序列表示为语义 ID 序列：

ℋu = {(s11, s12, s13), (s21, s22, s23), ..., (sn1, sn2, sn3)}
论文 §3.1

模型缩放

论文发现推荐模型的缩放同样遵循 Scaling Law。通过稀疏 MoE 架构，模型参数可以扩展到数十亿级别，而计算量（FLOPs）保持与稠密模型相当。消融实验（§5.3.2）显示增加 MoE 专家数量可显著提升 Recall@K。

3. OneRec: Session-Wise 列表生成

传统推荐方法采用逐点（Point-wise）预测（只预测下一个视频）。OneRec 提出会话级（Session-wise）生成：一次性生成用户一次请求对应的推荐列表（5-10 个视频），捕捉列表内部的依赖关系和多样性。

高质量 Session 筛选标准

  1. 用户实际观看的视频数 ≥ 5
  2. 用户在该 Session 上的总观看时长超过阈值
  3. 用户表现出互动行为（点赞、收藏、分享）

优化目标：𝒮 := ℳ(ℋu)，其中 ℋu 使用语义 ID 表示。

论文 §3.2

与传统方法对比

维度逐点预测Session-Wise
输出1 个 item5-10 个 item 列表
列表内依赖需手工规则模型自动学习
多样性控制后处理采样内生多样性
优势在于：Session-wise 让模型自学习最优的列表结构（哪些视频组合在一起），而逐点生成需要后续通过手工规则（MMR、DPP 等）来保证列表的连贯性和多样性。

4. OneRec: 迭代偏好对齐（IPA）

DPO Reward Model Self-Hard Negatives

Session-wise 训练让模型学会"好的 Session 是什么样"，但能否进一步对齐用户偏好？OneRec 提出迭代偏好对齐（Iterative Preference Alignment, IPA）：

挑战

推荐系统与 NLP 的 DPO 关键区别：推荐系统中每个用户请求只有一次展示机会，无法同时获得正负样本。因此需要 Reward Model 来模拟用户偏好并构造偏好对。

论文 §3.3

Reward Model 训练

以用户行为和 Session 特征为输入，预测用户对推荐 Session 的满意度。训练数据来自用户真实互动日志：

R(u, 𝒮) → score
# 正样本：用户观看完成的 session
# 负样本：用户跳过或未互动的 session
论文 §3.3.1

IPA 算法流程

/* 论文 Algorithm 1: Iterative Preference Alignment */
for epoch = t to T:
    for sample = 1 to N_sample:
        if rand() < rDPO:                   // 以概率 r_DPO 进行 DPO
            // 从 Beam Search 结果中选择 K 个候选
            candidates = beam_search(ℳepoch, ℋu, K)
            // 用 RM 对候选排序
            scores = RM(u, candidates)
            best, worst = select_extremes(scores)
            // DPO 更新——选择作为 chosen，最差作为 rejected
            loss = DPO_loss(ℳepoch, best, worst)
        else:
            // 继续标准 session-wise 训练
            loss = CE_loss(ℳepoch, session_data)
Self-Hard Negatives：IPA 不从随机采样中构造负样本，而是从 Beam Search 结果中选出最差的——这些是模型"以为很好但实际上用户不喜欢"的结果（Hard Negatives），DPO 通过拉开 best 和 worst 的概率来提升生成质量。

5. OpenOneRec: Itemic Token 嵌入

Qwen3-Embedding Residual K-Means 词表扩展

OpenOneRec 将每个物品编码为 LLM 词表中的一组可学习 token ID——Itemic Token。这是与 OneRec 论文中 RQ-VAE 类似的思路，但集成到了 Qwen3 的词表中。

<s_a_340> <s_b_6566> <s_c_5603>
→
拼入输入序列
<|sid_begin|>...(tokens)...<|sid_end|>

残差 K-Means 算法

编码

def encode(self, x):
    for l in range(n_layers):
        dist = torch.addmm(x_norm_sq + c_norm_sq,
                x, self.centroids[l].T, alpha=-2.0)
        code = dist.argmin(dim=-1)
        x = x - self.centroids[l][code]  # 残差
        out.append(code)
    return torch.stack(out, dim=1)      # [batch, L]
tokenizer/res_kmeans.py:27

解码

def decode(self, code):
    out = torch.zeros((code.shape[0], self.dim))
    for l in range(n_layers):
        c = code[:, l]
        out += self.centroids[l][c]    # 求和重建
    return out                         # [batch, dim]

词表扩展

在标准 Qwen3 tokenizer 基础上增加新 token。以 3 层、每层 8192 为例：

  • <s_a_0> ~ <s_a_8191> — 第 1 层码本索引
  • <s_b_0> ~ <s_b_8191> — 第 2 层码本索引
  • <s_c_0> ~ <s_c_8191> — 第 3 层码本索引
  • <|sid_begin|> / <|sid_end|> — 特殊边界标记

共 3 × 8192 + 2 = 24,578 个新 token，最终总词表对齐到 256 的倍数。

    --output_model_dir /path/to/Qwen3-0.6B_itemic \
    --itemic_layer_n 3 \

pretrain/tools/model_converter/expand_qwen3_vocab.py

脚本调用 tokenizer.add_tokens(new_tokens) + model.resize_token_embeddings(target_vocab_size)。新嵌入行随机初始化。itemic_id_range（如 [151669, 176246]）用于在训练中追踪哪些 token 是 itemic token。

Token 顺序必须严格固定：所有 s_a_* → s_b_* → s_c_* → 特殊标记，确保语义一致性。

Chunked Loss Computation

对于 ~175K 的词表，LM Head 投影矩阵极大。OpenOneRec 使用 ChunkedLossComputer 将序列切块，逐块计算 logits + 损失并手动累积梯度：

class ChunkedLossComputer:
    def forward_and_backward(self, input, labels):
        for i in range(0, seq_len, self.minibatch_size):
            input_chunk = input[:, start:end].detach().requires_grad_()
            logits_chunk = self.lm_head(input_chunk)
            loss_chunk = self.loss_fn(logits_chunk, labels_chunk)
            grads = torch.autograd.grad(loss_chunk_sum, tensors_to_grad)
            # 累积梯度到 lm_head 参数和输入隐状态

        if emb.weight.grad is not None:

Stage 2: On-Policy 蒸馏

知识蒸馏 通过 VeRL 框架，学生模型（OneRec-SFT）生成 rollout，教师模型（更大的 Qwen3 基座）提供 log-probability，最小化 KL 散度，恢复 SFT 中可能丢失的通用推理能力。
